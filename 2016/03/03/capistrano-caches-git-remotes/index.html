Capistrano caches git remotes

3 Mar, 2016 · Read in about 1 min · (126 Words)

Quick explanation on issue I solved for Capistrano

For a project I changed the ‘origin’ remote for the git repository. As I use capistrano for deployments I dutifully changed the git repository url in the deploy.rb file.

After making some changes and deploying them using the new ‘origin’ remote for the repository the webserver did not pick up on it. It seemed as if capistrano cached the old remote.

Apparently it did, the ‘repo’ git repository directory in the application directory contained a reference to the old remote. Removing the directory and deploying again through ‘cap production deploy’ made capistrano pick up on the changes.

I could also have gone into the ‘repo’ directory and changed the remote reference there but this works just as well.
